There is a difference between code that works and code that is good. The first is a bar that is embarrassingly easy to clear. The second requires thinking about the person who will read this in six months — which is usually you, having forgotten everything about the context in which you wrote it.

The engineers I most respect write code as if it is a form of communication. Not just to the machine, but to the next person who has to understand what the machine is being asked to do, and why.

I have been thinking about this more since starting to work on open-source projects. When the audience for your code is the entire internet, the pressure to write clearly is different. It is a useful pressure.
